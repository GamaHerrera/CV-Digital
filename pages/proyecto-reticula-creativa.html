Retícula Creativa
Arquitectura Digital y Swiss Brutalism

El Proyecto

Retícula Creativa es un colectivo y plataforma editorial enfocada en compartir conocimiento técnico avanzado sobre diseño gráfico y desarrollo web. El proyecto nació de la necesidad de crear un espacio digital que rechazara la sobrecarga visual y el código inflado de la web moderna, apostando por una experiencia pura, rápida y cruda.

El Objetivo

Diseñar y desarrollar un producto digital desde cero bajo la filosofía del Swiss Brutalism. El reto consistía en construir un sistema de diseño estrictamente tipográfico y modular, que priorizara tiempos de carga milimétricos y accesibilidad sin depender de plugins de terceros ni librerías pesadas.

El Reto y la Solución

La ejecución del proyecto se dividió en tres pilares fundamentales para garantizar que tanto la estética como el código funcionaran en perfecta sincronía técnica.

Pilar 01

Identidad Visual y UI (Swiss Brutalism)

El Enfoque: En lugar de interfaces amigables y redondeadas, se optó por un lenguaje visual técnico inspirado en terminales de comandos y planos arquitectónicos.

Ejecución: Se implementó una cuadrícula visible, paleta monocromática de alto contraste (negro absoluto y acentos de color neón) y tipografías monoespaciadas (JetBrains Mono). Elementos como formularios de contacto fueron rediseñados como interfaces de "Petición de Entrada" simulando una consola de sistema, filtrando así el tipo de usuarios e interacciones.

Pilar 02

Estandarización de Contenido (Open Graph System)

El Enfoque: Mantener la estética brutalista incluso cuando el contenido se compartiera en redes sociales externas (X, LinkedIn, WhatsApp), sin tener que diseñar cada portada manualmente.

Ejecución: Se creó un sistema de componentes dinámicos en Figma para las Open Graph Images (OGI). Se establecieron reglas de dirección de arte restrictivas: uso de efectos de Medio Tono (Halftone) o Dithering (1-bit) para las fotografías de los autores, tratándolas como metadatos visuales en lugar de retratos tradicionales, manteniendo los archivos finales por debajo de 300 KB.

Pilar 03

UX Técnica y Rendimiento (Performance)

El Enfoque: Eliminar la dependencia de bases de datos pesadas e integraciones de terceros que arruinaran las métricas de carga (Core Web Vitals).

  • Se eliminaron widgets sociales comerciales y se reemplazaron por la Web Share API nativa y enlaces HTML crudos.
  • Se optimizó la entrega de recursos transformando más de 5 MB de cargas gráficas al formato WebP.
  • Se diseñó un sistema de métricas asíncrono (Optimistic UI) utilizando localStorage para guardar artículos en caché (favoritos locales) y llamadas a una API ligera para la reputación global, usando caracteres ASCII ([+], [✔]) en lugar de iconografía vectorial pesada.

Galería de Sistema

Resultados y Métricas

Arquitectura Vainilla

100% construido con HTML, CSS y Vanilla JS. Cero frameworks de JavaScript, cero bloatware que impida la experiencia directa y cruda.

Despliegue Continuo (CI/CD)

Transición de despliegue manual a sincronización automatizada vía SFTP, reduciendo a cero el margen de error humano en actualizaciones de código.

Web Performance

Optimización profunda de métricas de carga (FCP y LCP) mediante políticas estrictas de compresión, minificación y control de caché en servidor NGINX.
